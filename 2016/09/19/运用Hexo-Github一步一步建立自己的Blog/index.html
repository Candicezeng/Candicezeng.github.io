运用Hexo+Github一步一步建立自己的Blog

发表于 2016-09-19   |   分类于 Hexo   |     |  

用了两个晚上，终于有了自己的博客了。感谢大懒虫童鞋的教程，感谢nexT主题的贡献者们！在此记录自己创建博客的历程，希望以后想要建站的童鞋们少走弯路！

在GitHub上建立博客站点

注册GitHub

在GitHub上注册一个账号，注意牢记自己注册时使用的邮箱。GitHub的通知，和密码的找回都是通过邮箱完成的。如下图：

填入你的username(用户名)，Email Address(邮箱地址)，Password(密码)就申请成功了。申请成功后别忘了验证邮箱哟~

在GitHub上建立仓库

在此运用自己的GitHub用户名建立一个形如username.github.io这样的可访问的站点（注意：每个用户名下只能建立一个）如下图所示：


注意此处建立的new repository的名称一定要输入自己的账号名，例如你的账号是xiaoming,则应在Repository name处输入xiaoming.github.io

搭建环境

安装软件

Node.js

https://nodejs.org/en/
下载完成后根据提示一步一步安装就好

Git

下载并安装Git
https://git-scm.com/download

GitHub for Windows

https://desktop.github.com/
下载Github for Windows。若你用的别的操作系统，则安装对应操作系统版本。
安装好之后使用GitHub for Windows登录你的GitHub

点击齿轮>-点击Options进入登录界面

点击add account后输入自己的账户密码即可

配置SSH

为了让本地git项目与远程GitHub建立建立联系，我们打开Git Shell,然后输入以下命令：

ssh -T git@github.com

如下图：

如果得到如下类似的反馈：

The authenticity of host ‘github.com (207.97.227.239)’ can’t be established. RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48. Are you sure you want to continue connecting (yes/no)?

只要输入yes就配置成功啦~

使用Hexo创建博客

Hexo 安装

在git shell中依次输入：

cd /
npm install hexo-cli -g

在此cd /作用是返回根目录，用不用cd /全看你啦~
出现如图所示情况则安装好了：

Hexo 部署

在Shell中输入命令：

cd /
hexo init Hexo

则可部署成功。在此你可以自己定义文件名，形如 hexo init [文件名]。
部署成功后，hexo会自动在目标文件夹创建博客网站所需的所有文件。此时可以通过输入以下命令，在本地进行预览：

hexo generate
hexo server

预览的前提是进入到刚才创建的Hexo文件夹中。在此也可简写命令，如：

hexo g
hexo s

系统若出现提示，则点击允许。命令允许完成后就可打开浏览器，输入http://localhost:4000/ (默认端口为4000)，就可看到最原始的博客了。以后再发表博文之前，也可以通过hexo server在本地进行预览。
博客效果图：

至此，个人博客的框架就搭建好了。按Ctrl+C就可停止预览。

将本地文件部署到GitHub

###修改_config.yml文件
在Hexo文件夹下找到_config.yml文件，并用文本编辑器打开。找到其中的deploy标签，修改为如下所示形式（冒号后面记得加上一个空格哟！）：

###安装deployer git
在Git Shell中输入以下命令：

npm install hexo-deployer-git –save

再依次输入以下命令：

hexo clean
hexo generate
hexo deploy

由此，个人博客就已经部署到GitHub上啦。你可以通过your_user_name.github.io(就是你新建的那个仓库名称)，就可以看到你的个人博客了~(≧▽≦)~
感谢大懒虫童鞋的教程，上面的搭建过程完全参考大懒虫童鞋的教程，如果你对搭建过程还有疑问可以参考大懒虫童鞋的博客：github+hexo建立你的第一个博客

发表博文

总算把博客搭建好啦，现在回到我们的重点–写博客上吧！

新建博文

在Git Shell中输入以下命令:

hexo new “文章题目”

命令执行后，Hexo\source_posts目录中就会添加一个以你刚刚命名的文档为标题的md文件，这就是我们刚才新建的博客文档了。
同时，我们也可以直接进入Hexo\source_posts中新建一个md文档。这也是新建博文的方法。

新建页面

输入以下命令：

hexo new page “页面名称”

命令执行后页面就新建好了。在Hexo/source目录中会增加一个以你新建页面为名的文件夹，里面包含一个index.md文件。
新建的博文是显示在单个文章页面里的。新建的页面则会作为单个页面显示在博客首页中，比如我所新建的如下页面：

图中的分类、标签是我新建的页面，点击进去后都为单个的页面。

写博客

用文本编辑器打开新建的博文就可以写博文啦，如下图：

新建的页面和我的可能有所不同，没有tags和categories两项
在’- - -‘后就可以书写博客正文啦。
由于我们的博文用的是Markdown语言编写的，所以你需要一个好用的Markdown编辑器。好用的Markdown编辑器有很多，大家可以自行google。我习惯于sublime+mahua结合来编写博文。在此推荐一下MaHua这个在线编辑器，非常方便。
如果你觉得不知从何入手Markdown的话，可以参考Markdown的教程http://www.markdowntutorial.com/

发博文

终于到了激动人心的发表时刻啦。
依旧是在Git Shell中，我们输入命令：

hexo clean(若不想删除保存于本地的已经发布的信息，重新生成新文件，则不需要输入此行)
hexo generate
(本地预览则输入hexo sever)
hexo deploy

快捷命令：

hexo g == hexo generate
hexo s == hexo server
hexo d == hexo deploy
hexo n == hexo new
hexo d -g #组合使用

刷新个人博客，你刚刚写的博客就已经发布啦~

使用NexT主题

hexo的主题是我非常喜欢的一个部分。hexo的主题非常丰富，关键是非常好看！在此当然要安装一个自己喜欢的主题啦！

选择主题

刚刚我们生成的博客使用的是Hexo的默认主题：Landscape。这个主题看第一眼其实还行，但当我打开hexo的主题页面，仿佛打开了新世界的大门，再一比较就发现landscape实在太low了！因此，我决定找寻一款自己喜爱的主题。
hexo有很多主题可以供选择，我多番比较，参考知乎有哪些好看的Hexo主题，最后选择了我现在使用的这个主题：NexT。这个主题简洁明朗，并且作者iissnan和其他贡献者保持着主题的更新，主题的功能非常全面，用起来不要太方便！
当然每个人喜好不同，选择你所钟爱的主题吧。如果你也喜好NexT主题的话，那就让我们一起愉快的玩耍吧！

安装NexT主题

首先在这里介绍NexT主题的网站，安装、添加内容、修改设置等各个方面的内容都可以在NexT的网页中找到：http://theme-next.iissnan.com/
网页中对于安装这个主题的细节、添加配置、第三方服务等描述的非常具体，在此我就不详述了。只大致介绍一下通用的主题安装流程。

下载NexT主题

输入以下命令：

cd hexo
git clone https://github.com/iissnan/hexo-theme-next themes/next

启用主题

在Hexo目录的_config.yml中找到theme字段，并将其值改为next。如下：

theme:next

到此，NexT主题就安装完成了。下一步我们将验证主题是否正确启用。在切换主题后、验证之前最好使用
hexo clean
来清除Hexo的缓存。

验证主题

首先启用Hexo本地站点，并开启调试模式

hexo s –debug

在服务启动的过程，注意观察命令行输出是否有任何异常信息，如果你碰到问题，这些信息将帮助他人更好的定位错误。当命令行中出现：

INFO Hexo is running at http://0.0.0.0:4000/. Press Ctrl+C to stop.

此时，用浏览器访问本地站点，检查站点是否正确运行。若如下图所示：

恭喜你，主题已经设置成功了！

主题设定

你可以根据NexT官方网站对主题进行各项设定，自由的折腾吧！
