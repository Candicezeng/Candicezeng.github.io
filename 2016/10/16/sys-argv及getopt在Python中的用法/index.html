sys.argv及getopt在Python中的用法

发表于 2016-10-16   |   分类于 Python   |     |  

sys.argv的用法

sys.argv[]的作用是获取命令行的参数，sys.argv[0]代表的是文件本身的路径。从sys.argv[1]开始是文件执行的参数。废话不多说，先上例子。
使用sys.argv的例子：
在此，我建立一个test.py的Python文件，内容如下：

[Python]
import sys
print ‘Number of arguments:’, len(sys.argv), ‘arguments.’
print ‘Argument List:’,str(sys.argv)

现在我们在terminal中运行test.py.在terminal中输入：

python test.py arg1 arg2 arg3

这样我们会得到：

Number of arguments:4 arguments.
Argument List: [‘test.py’,’arg1’, ‘arg2’, ‘arg3’]

这里的四个arguments分别是：test.py, arg1,arg2,arg3。其中test.py为sys.argv[0],arg1为sys.argv[1]，以此类推。那么这样的参数有什么作用呢？在接下来的getopt中会体现出其作用。

getopt用法

在运行程序的时候可能会需要根据不同的条件，输入不同的命令行选项来实现程序不同的功能。Python为我们提供了getopt来实现选项的解析，从而执行不同的选项实现程序。在Python中支持短选项和长选项两种格式。短选项格式为”-“加上单个字母，长选项格式为”–”加上一个单词。

取得命令行参数

在此我们就要使用刚才说到的sys.argv命令了。在python中创建test.py文件，并写入：
import os
print sys.argv

然后在terminal中写下任意参数，如：
python test.py -o input -h output –help me file1 file2
其结果为：
[‘test.py’, ‘-o’,’input’,’-h’,’output’,’–help’,’me’,’file1’,’file2’]
所有命令行都以空格为分隔符，保存在了sys.argv列表中。其中第一个元素为文件的路径。

选项的写法要求

对于短选项：’-‘后紧跟一个选项字母，如果有附加的参数则可紧跟在选项字母后，也可用空格与选项字母隔开。参数的长度不限，可以使用引号。个人比较喜欢空格后接参数的写法，一目了然。下面列举一些正确的写法：
-o
-oa
-oaaaa
-o aaaa
-o “abcd”
对于长选项，在”- -“后则需紧跟一个单词。若还有附加参数，则需在选项后紧跟一个”=”，再加上参数。注意：”=”号前后都不能有空格。正确的写法如下：
–help=inputfile

运用getopt解析命令行

运用getopt解析命令行大致分为三步：

  • 引入getopt，sys模块
  • 分析命令行参数

  • 处理解析结果
    还是上代码吧。
    第一步：

    import sys, getopt
    argv=sys.argv[1:]

    此处另argv变量为命令行参数list,去掉第一个参数（文件路径）这样可以方便处理命令参数
    第二步：

  • 此处处理程序的函数为getopt(),因为是直接使用import导入了getopt模块，因此要限定函数为getopt函数。

  • 使用短选项分析字符串”h:o”。在”h”后加入”:”号表示h后应附带一个参数。而”o”后没有引号则表示此选项只是一个开关，当出现此选项时开关打开。
  • 使用长选项分析字符串[“ifile=”,”ofile”]。同理，长选项后跟等号表示该选项后带有一个参数，若不带参数则会返回错误。长选项没有带等号则表示该选项为一个开关
  • 使用getopt函数会返回两个list：opts和args。opts是分析出的格式信息，是一个两元组的列表，每个元素为（选项，附加参数）。如果没有附加参数，则附加参数项为空。args为不属于格式信息的剩余命令行参数。
    若在terminal中执行：
    python test.py -h t -o –ifile=out –ofile file1 file2

    则解析完成后后opts为：
    [(‘-h’,’t’),(‘-o’,’’),(‘–ifile’,’out’),(‘–ofile’,’’)]
    args为：
    [‘file1’,’file2’]
    第三步：主要是判断解析出的参数是否存在，若存在再做进一步处理。
